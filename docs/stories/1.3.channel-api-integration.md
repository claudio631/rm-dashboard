## Story

**As a** marketing team member,
**I want** the platform to connect to Indeed Ads, Google Ads, Meta Ads, Bing Ads, Reddit Ads, and Craigslist APIs,
**so that** campaign data can be pulled and managed from a single system.

## Acceptance Criteria

1. Service abstraction layer (`ChannelService` interface) with unified methods: `getCampaigns()`, `getCampaignMetrics()`, `getAccountInfo()`
2. Google Ads API integration returning campaign list and basic metrics (spend, clicks, impressions, conversions)
3. Meta Marketing API integration returning campaign list and basic metrics
4. Indeed Ads integration (API or scraping adapter) returning sponsored job data
5. Bing Ads (Microsoft Advertising) API integration returning campaign list and metrics
6. Reddit Ads API integration returning campaign data
7. Craigslist adapter (manual data entry or CSV import) since no official API exists
8. All API credentials stored securely via environment variables, never hardcoded
9. Connection status health check endpoint at `/api/channels/status` showing connected/disconnected per channel
10. Graceful error handling when a channel API is unavailable (other channels continue working)
11. TypeScript interfaces defined for all channel data models in `src/types/channels.ts`

## Dev Notes

### Architecture
- Pattern: Adapter pattern — each channel implements a common `ChannelService` interface
- Location: `src/services/channels/{channel-name}.ts`
- Types: `src/types/channels.ts` for unified data models
- Config: Channel credentials in `.env` (GOOGLE_ADS_CLIENT_ID, META_APP_ID, etc.)
- Depends on: Story 1.1 (project foundation), Story 1.2 (auth for API routes)

### Channel API Notes
- **Google Ads:** Requires OAuth2 + developer token + customer ID
- **Meta:** Requires app ID + app secret + access token + ad account ID
- **Indeed:** Limited API; may need CSV import fallback
- **Bing:** Import from Google Ads possible; uses Microsoft Advertising API
- **Reddit:** Self-serve API with OAuth2
- **Craigslist:** No API — manual CSV import or data entry

### Testing
- Test location: `tests/services/channels/`
- Use mocked HTTP responses (no live API calls in tests)
- Test the adapter pattern: mock channel returning data → verify unified format
