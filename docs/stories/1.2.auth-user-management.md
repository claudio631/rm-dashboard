## Story

**As a** marketing team member,
**I want** to log in securely and have role-based access to the platform,
**so that** only authorized Indeed Flex staff can access campaign data and tools.

## Acceptance Criteria

1. User authentication via email/password with secure session management
2. Role-based access control with at least two roles: `admin` (full access) and `marketer` (campaign operations)
3. Protected routes redirect unauthenticated users to login page
4. Session persists across page refreshes with secure token storage
5. Logout clears session and redirects to login
6. API routes validate authentication before processing requests
7. User profile page displays name, email, and role
8. Rate limiting on login endpoint (max 5 attempts per minute per IP)

## Dev Notes

### Architecture
- Auth: NextAuth.js with Credentials provider OR Supabase Auth
- Sessions: JWT-based for stateless auth
- Protected routes: Next.js middleware at `src/middleware.ts`
- Role storage: User record in database with `role` field
- Depends on: Story 1.1 (project foundation)

### Security
- Passwords must be hashed (bcrypt or argon2)
- CSRF protection enabled
- Secure, httpOnly, sameSite cookies for session tokens
- NFR4: API credentials encrypted at rest, never in client code

### Testing
- Test location: `tests/auth/`
- Mock authentication provider for unit tests
- Integration tests for login/logout flow
