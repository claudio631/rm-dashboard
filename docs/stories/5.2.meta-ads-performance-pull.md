## Story

**As a** recruitment marketing specialist,
**I want** Meta Ads campaign performance data pulled and stored automatically,
**so that** I can analyze impressions, clicks, spend, and conversions across Facebook and Instagram campaigns alongside Reddit and other paid channels.

## Complexity
**Estimate:** M (Medium) — 5 points
**Rationale:** Builds directly on Story 5.1 foundation (auth + base client already done). Adds 2 DB tables, 1 service with 2 methods, 2 API routes, minimal UI surface, and 4 tests. Meta Insights API is well-documented and returns rich data in a single call.

## Scope

**IN:**
- `meta_campaigns` DB table — campaign metadata (id, name, status, objective, daily_budget, lifetime_budget)
- `meta_ad_performance` DB table — daily metric snapshots (impressions, clicks, spend, reach, ctr, cpc, cpm, conversions, synced_at)
- `MetaAdsService` with `syncCampaigns()` and `syncPerformance(dateRange)` methods
- `POST /api/meta/sync` route — triggers on-demand sync, returns summary
- `GET /api/meta/performance` route — returns stored metrics with optional `?days=N` filter
- Performance summary card on `/settings/integrations` page (last sync time, total campaigns, last 7d spend)

**OUT (future stories):**
- Creating or pausing campaigns → Story 5.3
- Instagram organic monitoring → Story 5.4
- Full analytics dashboard integration → TBD
- Automated scheduled sync (cron) → TBD
- Ad set and ad level breakdowns → TBD

## Risks
- **Ad Account ID format:** `META_AD_ACCOUNT_ID` must use the `act_` prefix (e.g. `act_123456789`). The numeric-only ID from Meta Business Manager will NOT work with the API.
- **App Review for permissions:** If the Meta App is in Development Mode, `ads_read` only works for users added as App Testers or Developers. For production, App Review may be required — validate before starting Task 2.
- **Token expiry:** If the stored OAuth token has expired (60-day limit), sync will fail with a 190 error. The route should return a clear error message indicating re-auth is needed, not a generic 500.
- **Conversion tracking:** `actions` field (conversions) is only populated if a Meta Pixel or Conversions API is configured on the ad account. If not set up, the field returns 0 — this is expected, not an error.
- **Date range limits:** Meta Insights API allows maximum 37-month lookback. For daily sync default of 30 days, this is well within limits.

## Acceptance Criteria

1. `meta_campaigns` table exists via Drizzle migration with fields: `id` (Meta campaign ID, PK), `name`, `status`, `objective`, `daily_budget`, `lifetime_budget`, `currency`, `synced_at`
2. `meta_ad_performance` table exists via Drizzle migration with fields: `id` (auto PK), `campaign_id` (FK → meta_campaigns.id), `date`, `impressions`, `clicks`, `spend`, `reach`, `ctr`, `cpc`, `cpm`, `conversions`, `synced_at`
3. `MetaAdsService.syncCampaigns()` calls `GET /act_{account_id}/campaigns` with required fields, upserts all campaigns into `meta_campaigns`
4. `MetaAdsService.syncPerformance(startDate, endDate)` calls the Meta Insights API at `GET /act_{account_id}/insights` with `level=campaign`, `time_increment=1`, upserts daily rows into `meta_ad_performance`
5. `POST /api/meta/sync` triggers both `syncCampaigns()` and `syncPerformance` (last 30 days by default), returns `{ campaigns_synced: number, performance_rows: number, synced_at: string }`
6. `GET /api/meta/performance` returns stored rows from `meta_ad_performance` with optional `?days=N` query param (default: 30)
7. `/settings/integrations` Meta card updated to show: last sync timestamp, total campaign count, total spend last 7 days
8. `npm run typecheck` and `npm test` both pass

### Meta Marketing API — Endpoints

**List campaigns:**
```
GET https://graph.facebook.com/v21.0/act_{account_id}/campaigns
  ?fields=id,name,status,objective,daily_budget,lifetime_budget,currency
  &access_token={token}
```
Response:
```json
{
  "data": [
    {
      "id": "120200000000000000",
      "name": "Indeed Flex - Orlando - Hospitality",
      "status": "ACTIVE",
      "objective": "OUTCOME_LEADS",
      "daily_budget": "5000",
      "currency": "USD"
    }
  ],
  "paging": { "cursors": { "before": "...", "after": "..." } }
}
```
**Note:** `daily_budget` and `lifetime_budget` are returned as strings in **cents** (e.g. `"5000"` = $50.00). **Divide by 100 before storing** — the DB schema stores USD float (e.g. `50.0`), not cents.

**Performance insights (Insights API):**
```
GET https://graph.facebook.com/v21.0/act_{account_id}/insights
  ?level=campaign
  &fields=campaign_id,campaign_name,impressions,clicks,spend,reach,ctr,cpc,cpm,actions
  &time_increment=1
  &time_range={"since":"2026-03-07","until":"2026-04-07"}
  &access_token={token}
```
Response:
```json
{
  "data": [
    {
      "campaign_id": "120200000000000000",
      "campaign_name": "Indeed Flex - Orlando - Hospitality",
      "impressions": "12450",
      "clicks": "347",
      "spend": "89.43",
      "reach": "8210",
      "ctr": "2.787149",
      "cpc": "0.257780",
      "cpm": "7.183936",
      "actions": [
        { "action_type": "lead", "value": "12" },
        { "action_type": "offsite_conversion.fb_pixel_lead", "value": "12" }
      ],
      "date_start": "2026-04-06",
      "date_stop": "2026-04-06"
    }
  ],
  "paging": { ... }
}
```

**Key differences from Reddit Ads API:**
- `spend` is returned as a **string in USD** (e.g. `"89.43"`) — NOT microcurrency. Parse as float.
- `ctr` is returned directly (not computed from clicks/impressions)
- `actions` is a list — extract the value for `"lead"` action type for conversions
- Pagination via cursor (`after` param) — use `paging.next` to get all pages if campaigns > 25
- `date_start` and `date_stop` mark the day for `time_increment=1` daily breakdown

### DB Schema Target

```typescript
// meta_campaigns
export const metaCampaigns = sqliteTable("meta_campaigns", {
  id: text("id").primaryKey(),                         // Meta campaign ID
  name: text("name").notNull(),
  status: text("status").notNull(),                    // ACTIVE | PAUSED | ARCHIVED
  objective: text("objective"),                        // OUTCOME_LEADS | OUTCOME_TRAFFIC | etc.
  dailyBudget: real("daily_budget"),                   // USD (converted from cents)
  lifetimeBudget: real("lifetime_budget"),             // USD (converted from cents)
  currency: text("currency"),
  syncedAt: text("synced_at").notNull(),
});

// meta_ad_performance
export const metaAdPerformance = sqliteTable("meta_ad_performance", {
  id: integer("id").primaryKey({ autoIncrement: true }),
  campaignId: text("campaign_id").notNull(),           // FK to meta_campaigns.id
  date: text("date").notNull(),                        // "YYYY-MM-DD" from date_start
  impressions: integer("impressions").notNull().default(0),
  clicks: integer("clicks").notNull().default(0),
  spend: real("spend").notNull().default(0),           // USD, stored as float
  reach: integer("reach").default(0),
  ctr: real("ctr"),                                    // percentage (e.g. 2.787)
  cpc: real("cpc"),                                    // USD
  cpm: real("cpm"),                                    // USD per 1000 impressions
  conversions: integer("conversions").notNull().default(0),  // extracted from actions[type=lead]
  syncedAt: text("synced_at").notNull(),
});
```

Add unique index on `(campaign_id, date)` for upsert conflict target.

### Extracting Conversions from `actions`

```typescript
function extractConversions(actions: Array<{ action_type: string; value: string }> = []): number {
  const lead = actions.find(a => a.action_type === "lead");
  return lead ? parseInt(lead.value, 10) : 0;
}
```

### Pagination Handling

Meta responses paginate at 25 items by default. For accounts with many campaigns:
```typescript
let url = `${BASE_URL}/act_{account_id}/campaigns?fields=...&access_token={token}`;
const campaigns = [];
while (url) {
  const res = await fetch(url).then(r => r.json());
  campaigns.push(...res.data);
  url = res.paging?.next ?? null;
}
```
Follow the same pattern in `syncPerformance` — insights may paginate too.

### Rate Limits

Meta uses Business Use Case (BUC) rate limits — score-based, not simple req/min:
- Marketing API tier: ~200 points/hour for Standard access
- A single sync (30 days of daily data, ~5 campaigns) consumes minimal quota
- Rate limit info returned in response headers: `X-Business-Use-Case-Usage`
- If rate limited, API returns error code `17` — log and surface to user; do not auto-retry immediately
