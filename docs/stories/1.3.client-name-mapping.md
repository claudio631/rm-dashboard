## Story

**As a** recruitment marketing specialist,
**I want** the system to automatically match data across FHS, Indeed, and Tableau despite different client naming conventions,
**so that** I get a unified view without manually reconciling names.

## Acceptance Criteria

1. Client name mapping configuration page at `/settings/client-mapping`
2. Pre-populated mapping table with known aliases (e.g., "CORT" ↔ "Cort" ↔ "US-B2C-Industrial-Cort")
3. User can add, edit, and delete client name mappings
4. Campaign name parser extracts client, vertical, role, and market from Indeed naming convention: `{country}-B2C-{vertical}-{client}-{role}-{market}-{date}`
5. Data join engine matches: FHS requisitions ↔ Indeed campaigns (by campaign code or parsed name) ↔ Tableau funnel (by client + location)
6. Joined data stored in `joined_metrics` table with: req_id, campaign, client_normalized, location, spend, clicks, applies, rsvps, verified, booked, completed
7. Join results page shows match rate: how many rows matched vs unmatched, with list of unmatched items for review
8. Fuzzy matching suggestion: when an unmatched client name is close to a known one, suggest the mapping
