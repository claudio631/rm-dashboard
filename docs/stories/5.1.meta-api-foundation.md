## Story

**As a** recruitment marketing specialist,
**I want** a Meta (Facebook) API client layer integrated into this project,
**so that** I can pull ad performance data across Facebook and Instagram campaigns from the same platform I use for Indeed and Reddit.

## Complexity
**Estimate:** L (Large) — 8 points
**Rationale:** New OAuth2 flow (Facebook Login), long-lived token exchange, DB migration, base API client, 3 API routes, Settings UI page, and 4 tests. Mirrors Reddit 4.1 in scope but Meta token lifecycle is different (60-day long-lived vs 1-hour + refresh).

## Risks
- **Redirect URI mismatch in local dev:** `META_REDIRECT_URI` must be registered in Meta App settings — in local dev, add `http://localhost:3000/api/meta/callback` as an allowed redirect URI in Meta for Developers
- **App Review requirements:** Meta may require App Review for `ads_read` and `ads_management` permissions on a live app — for a business tool used only by the team, adding testers to the App in Meta for Developers dashboard bypasses this requirement
- **Long-lived token expiry:** Long-lived User Access Tokens expire after 60 days — token refresh requires re-initiating the OAuth flow (no silent refresh like Reddit). Consider System User Token as a non-expiring alternative (see Dev Notes)
- **Business Manager access:** The connected Facebook account must be an Admin or Advertiser in the Meta Business Manager that owns the ad account

## Acceptance Criteria

1. `MetaAuthService` handles Facebook Login OAuth2 Authorization Code flow using `META_APP_ID`, `META_APP_SECRET`, and `META_REDIRECT_URI` from `.env`
2. OAuth callback route at `/api/meta/callback` exchanges auth code for a short-lived token, immediately exchanges it for a long-lived User Access Token (~60 days), and stores it persistently (SQLite via existing Drizzle ORM)
3. `MetaApiClient` base class wraps all HTTP calls to the Graph API (`https://graph.facebook.com/v21.0/`) with the access token injected automatically as `?access_token={token}` query param
4. A `meta_tokens` table is created via Drizzle migration: `id`, `access_token`, `token_type`, `expires_at`, `scope`, `created_at`, `updated_at`
5. `/api/meta/auth` route initiates the OAuth flow (redirects to Facebook authorization URL with required permissions: `ads_read ads_management`)
6. `/api/meta/status` route returns connection status: `{ connected: boolean, scopes: string[], expires_at: string, days_until_expiry: number }`
7. A Settings page entry at `/settings/integrations` shows Meta connection status with "Connect Meta" / "Disconnect" button — includes a warning badge when token expires within 7 days
8. All Meta API errors are normalized into a consistent `MetaApiError` type with `status`, `code`, `error_subcode`, and `message`
9. Token expiry check on every API call — if expired, return a clear error prompting re-auth (Meta long-lived tokens cannot be silently refreshed without a new OAuth flow)
10. `npm run lint`, `npm run typecheck`, and `npm test` all pass

## Dev Notes

### Credentials (add to .env — do NOT hardcode)
```
META_APP_ID
META_APP_SECRET
META_REDIRECT_URI      # e.g. https://www.indeedflex.com/api/meta/callback
META_AD_ACCOUNT_ID     # format: act_XXXXXXXXXXXXXXXXX (with act_ prefix)
```

### OAuth2 Flow (Facebook Login — Authorization Code)

Meta uses standard 3-legged OAuth via Facebook Login:

1. User clicks "Connect Meta" → redirect to `https://www.facebook.com/dialog/oauth`
2. User approves → Facebook redirects to `META_REDIRECT_URI?code=XXX&state=YYY`
3. Backend exchanges code for short-lived token via GET:
   ```
   https://graph.facebook.com/v21.0/oauth/access_token
     ?client_id={META_APP_ID}
     &client_secret={META_APP_SECRET}
     &redirect_uri={META_REDIRECT_URI}
     &code={code}
   ```
4. Backend immediately exchanges short-lived token for long-lived token (~60 days) via GET:
   ```
   https://graph.facebook.com/v21.0/oauth/access_token
     ?grant_type=fb_exchange_token
     &client_id={META_APP_ID}
     &client_secret={META_APP_SECRET}
     &fb_exchange_token={short_lived_token}
   ```

**Important:** Long-lived tokens (60 days) cannot be silently refreshed — once expired, the user must re-authenticate via OAuth. There is no refresh_token equivalent. For a permanent solution, use a System User Token (see below).

### System User Token (Recommended for Production)

For server-to-server access without token expiry, use a System User Token:
1. Go to Meta Business Manager → System Users
2. Create a system user with Advertiser role on the Ad Account
3. Generate a token with `ads_read` and `ads_management` permissions
4. Store as `META_SYSTEM_USER_TOKEN` in `.env` — no OAuth flow needed

If System User Token is available in `.env`, `MetaAuthService.getValidToken()` should prefer it over the DB-stored OAuth token. This can be implemented as a fallback check: if `META_SYSTEM_USER_TOKEN` is set, use it and skip OAuth flow entirely. Scope both patterns in this story but leave System User as optional (no UI needed).

### Meta Graph API Base URLs
- **API base:** `https://graph.facebook.com/v21.0/`
- **Auth dialog:** `https://www.facebook.com/dialog/oauth`
- **Token endpoint:** `https://graph.facebook.com/v21.0/oauth/access_token`
- **Token debug:** `https://graph.facebook.com/debug_token` (useful for inspecting permissions)

### Required Permissions
| Permission | Purpose |
|-----------|---------|
| `ads_read` | Read campaigns, ad sets, ads, and performance insights |
| `ads_management` | Pause/resume campaigns (Story 5.3) |

**Note:** Unlike Reddit's account-level API access, Meta permissions are declared at the app level AND must be granted by the user during OAuth. The connected Facebook account must be an Admin or Advertiser in the Business Manager that owns `META_AD_ACCOUNT_ID`.
