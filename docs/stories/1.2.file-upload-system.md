## Story

**As a** recruitment marketing specialist,
**I want** to drag-and-drop my FHS, Indeed, and Tableau Excel exports into the app,
**so that** the system can ingest my daily data without manual copy-pasting into spreadsheets.

## Acceptance Criteria

1. Upload page at `/upload` with drag-and-drop zone accepting `.xlsx`, `.xls`, and `.csv` files
2. System auto-detects file type based on column headers: FHS (has `requisition_id`, `job_title`, `rsvps`), Indeed (has `Campaign`, `Impressions`, `Spend`), Tableau OB Funnel (has `Slice by.. 1`, `Worker Accounts Created`)
3. Upload supports multiple files simultaneously (drop all 3 at once)
4. After upload, displays file summary: filename, detected type, row count, column count, date range detected
5. Files are parsed client-side using SheetJS (xlsx) — no server upload of raw files needed
6. Parsed data is sent to API routes and stored in the database
7. Upload history page shows past uploads with date, type, row count, and status
8. Error handling: graceful message if file format is unrecognized or columns don't match expected schema
9. Loading indicator during parsing of large files (15K+ rows for FHS)

## Dev Notes

### Architecture Reference
- `docs/architecture/mvp-architecture.md` — File Parser Specifications section
- `docs/context/daily-tracking-spreadsheet.md` — exact column definitions for FHS and Indeed

### FHS Expected Columns
`last_updated`, `requisition_id`, `job_id`, `job_title`, `agency`, `client`, `location`, `pay_rate_min`, `pay_rate_max`, `pay_type`, `rsvps`, `target_rsvps`, `status`

### Indeed Expected Columns
`Campaign`, `Impressions`, `Clickthrough rate (CTR)`, `Clicks`, `Apply start rate (ASR)`, `Apply starts`, `Apply completion rate (ACR)`, `Applies`, `Apply rate (AR)`, `Spend`, `Cost per click (CPC)`, `Cost per apply start (CPAS)`, `Cost per apply (CPA)`, `Job Count`

### Tableau OB Funnel Expected Format
Pivot table: `Slice by.. 1` (client), `Slice by.. 2` (location), stage name in col 3, then date columns, `Grand Total` last column. Stages: Worker Accounts Created, 1st Role Verified, 1st OB Task Completed, Platform Verified, "Ready to Book" Estimate, 1st Shift Booked, 1st Shift Completed, + 7 CR% rows.

### Key: Tableau parser is the hardest
The Tableau export is a **pivot table** (stages as rows, dates as columns) not a flat table. The parser must handle this transposed format.
